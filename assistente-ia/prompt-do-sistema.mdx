O prompt do sistema é o componente mais crucial do seu assistente de IA. Ele define a personalidade, comportamento, conhecimento e capacidades do seu assistente. Pense nele como o "cérebro" e o "manual de treinamento" combinados.

## Início Rápido com Modelos

Para começar rapidamente:

1. Vá para as configurações do seu assistente
2. Encontre o campo do prompt do sistema
3. Clique no botão "Modelos" próximo
4. Escolha um modelo que corresponda ao seu caso de uso
5. Personalize-o para suas necessidades

## Suporte a Idiomas

Seu prompt do sistema pode ser escrito em qualquer idioma, independentemente da configuração de idioma falado:

- Escreva o prompt em seu idioma preferido
- Configure o idioma falado separadamente nas configurações do assistente
- A IA seguirá as instruções do prompt enquanto fala no idioma selecionado

Por exemplo:

- Prompt do sistema em inglês, idioma falado configurado para espanhol
- Prompt do sistema em alemão, idioma falado configurado para francês
- Prompt do sistema em chinês, idioma falado configurado para inglês

## Por que São Importantes

Seu prompt do sistema:

- Molda como a IA pensa e responde
- Define limites da conversa
- Fornece conhecimento essencial
- Controla comportamento e tom
- Determina o tratamento de casos extremos

## Componentes Principais

Um bom prompt do sistema deve incluir:

### 1. Papel e Identidade

```
Você é um representante de vendas profissional da [Empresa]. Você se especializa em [Produto/Serviço] e tem amplo conhecimento de nossas ofertas.
```

### 2. Estilo de Conversa

```
Mantenha um tom amigável e profissional. Use linguagem clara e concisa. Evite jargões técnicos a menos que seja especificamente solicitado.
```

### 3. Informações Principais

```
Nossos principais produtos são:
- Produto A (R$ X/mês): [recursos]
- Produto B (R$ Y/mês): [recursos]
Promoção atual: 20% de desconto nos primeiros 3 meses
```

### 4. Diretrizes Comportamentais

```
- Sempre verifique as informações do cliente antes de discutir detalhes da conta
- Transfira para um agente humano se o cliente parecer frustrado
- Não faça promessas sobre datas de entrega
```

### 5. Estrutura de Resposta

```
Quando perguntado sobre preços:
1. Primeiro entenda suas necessidades
2. Apresente opções de pacotes relevantes
3. Explique a proposta de valor
4. Compartilhe descontos aplicáveis
```

## Melhores Práticas

1. **Seja Específico**

   - Instruções claras geram melhores resultados
   - Inclua exemplos de boas respostas
   - Defina o que NÃO fazer

2. **Estrutura Importa**

   - Organize informações logicamente
   - Use marcadores e seções
   - Mantenha informações relacionadas juntas

3. **Teste e Itere**
   - Comece com um prompt básico
   - Teste vários cenários
   - Refine baseado nas gravações de chamadas
   - Adicione tratamento para casos extremos

## Erros Comuns

- **Muito Vago**: "Seja útil e profissional" (Não específico o suficiente)
- **Muito Rígido**: Criar scripts para todas as respostas possíveis (Reduz o fluxo natural)
- **Sobrecarga de Informações**: Incluir detalhes desnecessários
- **Falta de Diretrizes**: Não especificar como lidar com situações comuns

## Exemplo de Estrutura

```
# Papel e Propósito
[Defina quem é a IA e seus objetivos principais]

# Conhecimento Central
[Informações essenciais sobre produtos/serviços]

# Diretrizes de Conversação
[Como interagir com clientes]

# Padrões de Resposta
[Como lidar com situações específicas]

# Limitações e Fronteiras
[O que a IA NÃO deve fazer]
```

## Testando Seu Prompt

1. Faça chamadas de teste cobrindo:

   - Cenários comuns
   - Casos extremos
   - Situações difíceis
   - Várias personalidades de clientes

2. Revise e ajuste:
   - Ouça gravações das chamadas
   - Verifique precisão das respostas
   - Confirme consistência do tom
   - Teste retenção de conhecimento

## Dicas de Otimização

- Comece com o Motor Rápido para iterações rápidas
- Use gravações de chamadas para identificar lacunas
- Adicione exemplos de boas/más respostas
- Inclua tratamento para perguntas inesperadas

---

**Dica Profissional:** Seu prompt do sistema é um documento vivo - atualize-o regularmente baseado em experiências reais de chamadas e interações com clientes.
